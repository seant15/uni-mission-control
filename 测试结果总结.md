## 🏗️ 发现的架构

### OpenClaw 实际架构:

```
┌─────────────────────────────────────────┐
│ http://open.unippc24.com:9090           │
│ OpenClaw Control (前端)                  │
│ - 用户界面                               │
│ - SPA 应用                               │
└─────────────┬───────────────────────────┘
              │
              ▼ 前端通过 WebSocket 连接
┌─────────────────────────────────────────┐
│ ws://100.x.y.z:18789                    │
│ 或 http://127.0.0.1:18789               │
│ OpenClaw Gateway API (后端)              │
│ - 真正的 API 服务                        │
│ - 只在内部网络可访问                     │
└─────────────────────────────────────────┘
```

**关键发现**:
- **端口 9090**: 前端界面 (可公开访问)
- **端口 18789**: API 服务 (仅内部网络)
- **IP 100.x.y.z**: Tailscale VPN 私有地址

---

## ❗ 问题分析

### 问题: API 不可公开访问

OpenClaw Gateway API (端口 18789) 只能通过以下方式访问:

1. **本地访问**: `http://127.0.0.1:18789`
   - 只能在运行 OpenClaw 的服务器上访问

2. **Tailscale VPN**: `http://100.x.y.z:18789`
   - 只能在 Tailscale 网络内部访问
   - 需要 Tailscale 客户端

3. **前端代理**: OpenClaw Control 前端已经连接
   - 前端在浏览器中运行
   - 通过 WebSocket 连接到 API

---

## 💡 解决方案选择

### 方案 A: 直接使用 OpenClaw Control ⭐ 推荐

**说明**:
- 不需要集成
- 直接使用现有的 OpenClaw Control

**实施**:
```typescript
// 在 Mission Control 中添加链接
<a href="http://open.unippc24.com:9090" target="_blank">
  打开 OpenClaw Control
</a>
```

**优势**:
- ✅ 不需要任何配置
- ✅ 不需要 API 访问
- ✅ 不需要 CORS 设置
- ✅ 立即可用

**劣势**:
- ❌ 需要打开两个网页

---

### 方案 B: 通过 iframe 嵌入

**实施**:
```typescript
// 在 Mission Control 中嵌入 iframe
<iframe
  src="http://open.unippc24.com:9090"
  width="100%"
  height="800px"
/>
```

**优势**:
- ✅ 在 Mission Control 中直接显示
- ✅ 不需要打开新窗口
- ✅ 不需要 API 访问

**劣势**:
- ⚠️ 可能有 iframe 安全限制
- ⚠️ 不能完全控制样式

---

### 方案 C: Tailscale VPN 访问 (复杂)

**要求**:
1. Mission Control 服务器加入 Tailscale 网络
2. 配置 Tailscale 客户端
3. 使用内部地址 `http://100.x.y.z:18789`

**实施**:
```env
VITE_OPENCLAW_GATEWAY_URL=http://100.x.y.z:18789
```

**优势**:
- ✅ 直接访问 API
- ✅ 完全控制

**劣势**:
- ❌ 需要 Tailscale 配置
- ❌ 只能在服务器端访问 (不能在浏览器)
- ❌ 用户浏览器无法直接访问

---

### 方案 D: 保持独立 + Mock 数据 ⭐ 当前状态

**说明**:
- Mission Control 使用 Mock 数据
- 需要 OpenClaw 时,打开 OpenClaw Control

**当前配置**:
```env
VITE_USE_MOCK_DATA=true
VITE_OPENCLAW_GATEWAY_URL=http://open.unippc24.com:9090
```

**优势**:
- ✅ 简单清晰
- ✅ 各司其职
- ✅ 可以演示所有功能

**劣势**:
- ⚠️ 数据不是实时的
- ⚠️ 需要告诉用户这是 Mock 数据
